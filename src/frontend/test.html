Language Tandem App - Test Mode

This page provides instructions for testing the Language Tandem App without a backend server. The app includes mock data generation and local storage to enable full testing of the frontend functionality.

Note: In a production environment, this application would connect to the FastAPI backend described in the requirements. For testing purposes, all API calls will use mock data instead.

Testing Instructions

  1. Launch the application by opening index.html in your browser.
  2. Sign up/login with any email address (e.g., test@example.com). The app will create a mock user.
  3. Complete your profile by setting your native language, learning language, and proficiency level.
  4. Explore the features:
    • Generate exercises of different types
    • Track your progress on the Progress page
    • Explore cultural content for your target language

Mock Data Generation

The application automatically generates the following types of mock data:

  • User data - Created during signup and stored in localStorage
  • Exercises - Generated when accessing the Exercises page or clicking "Generate Exercises"
  • Progress data - Created when accessing the Progress page
  • Cultural content - Generated when accessing the Cultural page or requesting new content

Testing API Connection

If you have the backend running, you can configure the app to use it:

  1. Start the backend server according to the instructions in the backend README
  2. Make sure the server is running at http://localhost:8000
  3. The frontend will automatically try to connect to this URL (configured in js/config.js)
Note: If the backend is not available, you'll see warning notifications that the app is using mock data instead. This is expected behavior.

Local Storage

The application uses localStorage to persist data between sessions. If you want to reset all data, you can clear localStorage:

// In your browser's console:
localStorage.clear();

Troubleshooting

If you encounter any issues:

  1. Check your browser console for error messages
  2. Try clearing localStorage and restarting the application
  3. Ensure all JavaScript files are properly loaded (check the Network tab in Developer Tools)
  4. If testing with a backend, verify that CORS is properly configured
